## Project Overview

ChapterCheck is an audiobook library management application with a **Next.js web app** and a **native SwiftUI iOS app**, sharing a Convex real-time backend. It features full-text search, author/series organization, custom bookshelves, user profiles, and an integrated audio player with multi-part book support and offline playback.

## Monorepo Structure

Turborepo + Yarn 4 workspaces:

- `apps/web/` — Next.js 15 web app (TypeScript, Tailwind CSS, shadcn/ui)
- `apps/ios/` — Native SwiftUI iOS app (Swift, XcodeGen)
- `packages/convex-backend/` — Convex schema, queries, mutations, actions
- `packages/shared/` — Zod validations, hooks, utils, types (shared by web)
- `packages/tailwind-config/` — Shared Tailwind theme, `cn()`, color tokens

Key files: `convex.json` at root points to `packages/convex-backend/convex`. `apps/web/.env.local` symlinks to root `.env.local`.

## Development Commands

```bash
# Start development (run both in parallel)
npm run dev           # Next.js dev server (port 3000)
npx convex dev        # Convex backend (separate terminal)

# Quality checks (web)
npm run lint          # ESLint (--max-warnings=0)
npm run type-check    # TypeScript strict mode
npm run format:check  # Prettier check

# Testing (web)
npm run test          # Vitest watch mode
npm run test:run      # Single run (CI)

# Convex
npx convex codegen    # Regenerate types after schema changes
npx convex deploy     # Deploy backend to production

# Deployment (web)
npm run cf-deploy     # Build and deploy to Cloudflare Workers

# iOS
# Open apps/ios/ChapterCheck.xcodeproj in Xcode
# Or regenerate project: cd apps/ios && xcodegen generate
```

## Architecture

### Tech Stack

| Layer            | Web                                       | iOS                               |
| ---------------- | ----------------------------------------- | --------------------------------- |
| **UI**           | React 19, Tailwind CSS, shadcn/ui + Radix | SwiftUI, @Observable              |
| **Framework**    | Next.js 15 (App Router)                   | Native iOS 26 (XcodeGen)          |
| **Backend**      | Convex (real-time DB + serverless)        | Same Convex backend               |
| **Auth**         | Clerk (webhook sync)                      | Clerk iOS SDK (JWT → Convex)      |
| **Storage**      | Cloudflare R2 (signed URLs)               | Same R2 via presigned URLs        |
| **Deployment**   | Cloudflare Workers + Pages                | Xcode / TestFlight                |
| **Dependencies** | npm packages                              | ConvexMobile 0.7.0, ClerkKit 1.0+ |

### Database Schema (packages/convex-backend/convex/schema.ts)

Tables: `users`, `books`, `authors`, `series`, `bookAuthors` (many-to-many with roles), `audioFiles`, `storageAccounts`, `shelves`, `shelfBooks`, `reviews`, `listeningProgress`, `genres`, `bookGenreVotes`, `userPreferences`

Key features:

- Full-text search indexes on name/title fields
- Series ordering supports decimals (e.g., 2.5 for novellas)
- Storage accounts isolate R2 paths per user

## Web App (`apps/web/`)

### Key Directories

- `app/` — Next.js App Router pages and layouts
- `components/` — React components organized by feature (audio/, books/, authors/, ui/)
- `hooks/` — Custom React hooks (audio playback, uploads, pagination, etc.)
- `contexts/` — React Context providers (AudioPlayerContext)

### Data Flow

- Convex provides real-time data via `useQuery()` hooks
- Mutations through `useMutation()` trigger automatic UI updates
- Clerk auth state synced to Convex `users` table via webhook
- R2 files accessed via signed URLs generated by Convex actions

### Audio Player

Global state in `AudioPlayerContext` handles playback across the app. Supports multi-part books, playback rate persistence, and both mini (fixed bottom) and expanded player views.

### Code Conventions

- **Strict TypeScript:** No `any` types, all files use `.ts`/`.tsx`
- **Import sorting:** Automatic via eslint-plugin-simple-import-sort
- **Styling:** Tailwind CSS only, use `cn()` from `packages/tailwind-config` for conditional classes
- **Components:** Use shadcn/ui components from `components/ui/`
- **Forms:** React Hook Form + Zod validation schemas from `packages/shared/validations`

## iOS App (`apps/ios/`)

### Key Directories

```
ChapterCheck/
├── App/              — Entry point (ChapterCheckApp.swift)
├── Auth/             — Clerk auth flow (AuthGateView, SignInView, ClerkConvexAuthProvider)
├── Config/           — Environment (dev/prod URL switching)
├── Models/           — Decodable structs matching Convex schema
├── Repositories/     — Convex data access layer (subscribe, mutate, action)
├── Services/         — ConvexService, DownloadService, DownloadManager, NetworkMonitor
├── Audio/            — AudioPlayerManager, NowPlayingManager, StreamURLCache, VoiceBoost
├── Features/         — Feature modules (Home/, Library/, BookDetail/, Player/, etc.)
├── Navigation/       — AppDestination enum, MainTabView (owns player + download state)
├── Theme/            — ThemeManager, AccentColorToken
└── Utilities/        — Haptics, TimeFormatting
```

### Architecture Patterns

- **Views → ViewModels → Repositories → ConvexService** (clean layered architecture)
- **@Observable + @MainActor** on all ViewModels for reactive SwiftUI updates
- **Combine publishers** for Convex real-time subscriptions (`ConvexService.subscribe()`)
- **Actor isolation** for thread-safe caching (StreamURLCache, ImageRepository, DownloadService)
- **Environment injection** for singletons: AudioPlayerManager, DownloadManager, ThemeManager

### Data Flow

- Repositories wrap `ConvexService.shared` — subscribe for real-time, mutation/action for writes
- ViewModels hold `AnyCancellable` subscriptions and expose `@Observable` state
- Convex function paths follow: `"module/queries:functionName"` or `"module/mutations:functionName"`
- All numeric params passed as `Double` (Convex `v.number()` expects float64, not Int)

### Audio Player

`AudioPlayerManager` (@Observable, @MainActor, ~1100 lines) orchestrates AVPlayer, NowPlayingManager (lock screen/Control Center), AudioSessionManager (interruptions), and StreamURLCache (presigned R2 URLs with 50min TTL). Supports multi-part auto-advance, playback rate, sleep timer, voice boost, momentum skipping, smart rewind. Progress saved every 10s + on pause/background.

### Offline Support

- `DownloadManager` (@Observable) tracks download state, delegates to `DownloadService` (actor)
- Files stored in `Documents/Downloads/{bookId}/` with `manifest.json`
- `OfflineProgressQueue` (actor) queues progress saves when offline, flushes on network restore
- `NetworkMonitor` (NWPathMonitor) tracks connectivity
- When offline: same UI gracefully degrades — stale Convex data stays visible, offline banner shown, features like search/shelves disabled. Downloaded books populate Continue Listening. Views recover to live subscriptions when network returns.

### Mini Player Layout

The mini player is a ZStack overlay in `MainTabView` positioned at the bottom. All scrollable views add 80pt bottom padding (ScrollView: `Spacer().frame(height: 80)`, List: `Color.clear.frame(height: 80)` with hidden row styling) to prevent content from being hidden.

### Code Conventions

- **Swift 5.10**, iOS 26 deployment target
- **@Observable** macro for all ViewModels (not ObservableObject)
- **@MainActor** on ViewModels and Services that touch UI state
- **Actor** for thread-safe state (caches, file I/O)
- **Decodable structs** for models — make non-essential fields optional for flexible Convex responses
- **Feature folders** group View + ViewModel + subviews together
- **No storyboards** — 100% SwiftUI with programmatic navigation via NavigationStack + AppDestination enum

## Environment Variables

Copy `.env.example` to `.env.local` at root. Required:

- `CONVEX_DEPLOYMENT`, `NEXT_PUBLIC_CONVEX_URL`
- Clerk keys (`NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`)
- R2 credentials (`R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, etc.)

iOS uses `Config/Environment.swift` with `#if DEBUG` for dev/prod Convex URL and Clerk key switching.
